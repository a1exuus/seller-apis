# Синхронизация товаров: Casio → Маркетплейсы

## Что делает этот инструмент

Этот проект автоматически обновляет **цены** и **остатки товаров** в ваших магазинах на маркетплейсах **Ozon** и **Яндекс Маркет**. Данные берутся из актуального файла поставщика часов **Casio (Timeworld)**.

## Зачем это нужно

| Без автоматизации | С этим скриптом |
|-------------------|-----------------|
| Менеджер вручную проверяет файл с остатками | Скрипт сам скачивает актуальный файл |
| Вручную меняет цены в личных кабинетах | Цены обновляются автоматически на обеих площадках |
| Риск продать товар, которого нет в наличии | Остатки синхронизируются точно |
| Тратится несколько часов в неделю | Процесс занимает минуты |

## Как это работает
Файл Casio → Скрипт → Ozon
           (сверка) → Яндекс Маркет

1. **Скачивает** актуальный файл с остатками с сайта поставщика (Timeworld)
2. **Сравнивает** товары из файла с товарами в ваших магазинах
3. **Обновляет** цены и количество товаров на обоих маркетплейсах

## Что именно обновляется

| Данные | Откуда берутся | Куда загружаются |
|--------|----------------|------------------|
| Цена товара | Файл Casio | Карточка товара на Ozon и Яндекс Маркет |
| Количество | Файл Casio | Остатки на складах маркетплейсов |
| Отсутствие товара | Если товара нет в файле | Остаток = 0 (нет в продаже) |

## Что нужно для работы

### Для Ozon (seller.py)
- **Client ID** — идентификатор вашего магазина в Ozon
- **Seller Token** — ключ доступа к API Ozon

### Для Яндекс Маркета (market.py)
- **Market Token** — OAuth-токен для доступа к API Яндекс Маркета
- **Campaign ID** — идентификаторы кампаний (FBS и DBS)
- **Warehouse ID** — идентификаторы складов (FBS и DBS)

Эти данные хранятся в защищённом месте и не передаются третьим лицам.

## Как часто запускать

Рекомендуется запускать синхронизацию:
- **Ежедневно** — для актуальных остатков
- **При изменении цен** у поставщика
- **Утром** — перед началом активного периода продаж

## Важные моменты

| Ситуация | Как обрабатывается |
|----------|-------------------|
| Товар есть в файле, но нет на маркетплейсе | Не загружается (требуется создание карточки) |
| Товар есть на маркетплейсе, но нет в файле | Остаток ставится в 0 |
| Количество ">10" у поставщика | Устанавливается 100 шт. для продажи |
| Количество "1" у поставщика | Остаток ставится в 0 (последняя единица не продаётся) |

## Особенности работы с Яндекс Маркетом

Яндекс Маркет поддерживает две схемы работы:

| Схема | Описание |
|-------|----------|
| **FBS** (Fulfillment by Seller) | Продавец хранит и отправляет товары сам |
| **DBS** (Delivery by Seller) | Продавец полностью отвечает за доставку |

Скрипт обновляет остатки и цены для **обеих схем** одновременно.

## Если что-то пошло не так

Скрипт сообщает об ошибках:
- Проблемы с интернет-соединением
- Превышено время ожидания ответа от сервера
- Ошибки доступа к данным (неверные токены)

В таких случаях нужно проверить:
1. Подключение к интернету
2. Актуальность токенов доступа
3. Доступность сайтов поставщика и маркетплейсов

## Преимущества использования

✅ **Экономия времени** — не нужно вручную обновлять сотни товаров  
✅ **Точность данных** — исключаются человеческие ошибки  
✅ **Актуальность** — покупатели видят реальные остатки и цены  
✅ **Масштабируемость** — работает с любым количеством товаров  
✅ **Две площадки** — одна настройка для Ozon и Яндекс Маркета
